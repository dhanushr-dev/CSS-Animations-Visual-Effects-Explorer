Introduction

Why CSS motion & visuals matter, and what you'll learn.

✨ What are CSS Animations?

CSS animations let elements change from one style to another smoothly, without JavaScript. You define @keyframes that describe the styles at various points, then attach them with the animation shorthand. The browser handles every frame on the compositor thread, so they stay smooth even under load.

🎯 Why use Animations?

Motion guides attention, communicates state changes, gives interfaces personality, and makes products feel polished. Used with restraint, animations improve usability: a button that subtly grows on hover tells the user "I'm clickable" instantly.

🖌️ What are CSS Visual Effects?

Visual effects cover everything that changes an element's appearance without altering layout: shadows, gradients, filters, clip-paths, backdrop filters, and transforms. Combined with motion they create depth, realism, and the "wow" factor of modern UI.

📚 Learning Objectives

By the end of this guide you will be able to:

  • Transform, transition & animate any element
  • Build reusable timing & keyframe patterns
  • Apply shadows, gradients & filters confidently
  • Ship glassmorphism, clip-paths & 25+ animation effects

CSS Transform

Move, scale, rotate & skew elements in 2D and 3D space. Hover any demo.

💡
Hover the demo boxes. Transforms never trigger reflow — the browser paints them on the GPU, so they're perfect for performant motion.

transform The master property

Definition
Applies a 2D or 3D transformation to an element.
Syntax
transform: <function>(...);
Values
none · translate() · scale() · rotate() · skew() · matrix() · perspective() and their 3D variants.
Hover me
.box { transform: matrix(1.2, 0.2, 0.3, 1.2, 20, 10); }

✅Tip: Chain transforms left-to-right: transform: rotate(45deg) scale(1.2);

CSS Transition

Smoothly interpolate between two states — usually triggered on hover/focus.

transition shorthand

Definition
Animates changes to CSS properties over a duration.
Syntax
transition: property duration timing-function delay;
Example
transition: transform .35s ease .1s;

transition-property

Which property animates. Hover to compare.

all
background
transform
.tr-bg { transition-property: background; }
.tr-bg:hover { background: var(--grad-warm); }

transition-duration

How long it lasts. 0.2s vs 0.6s vs 1.2s:

0.2s
0.6s
1.2s
.tr-d1 { transition: transform .2s var(--ease); }
.tr-d3 { transition: transform 1.2s var(--ease); }

transition-delay

Wait before it starts. 0.2s · 0.6s · 1s:

.2s
.6s
1s
.tr-delay2 { transition: transform .4s ease .6s; }
.tr-delay2:hover { transform: translateY(-24px); }

transition-timing-function & easing curves

The mathematical curve that controls acceleration. Hover each track and watch how the ball accelerates.

ease
linear
ease-in
ease-out
ease-in-out
step-start
step-end
steps(8)
cubic-bezier(.68,-.55,.27,1.55)
/* built-in */
transition-timing-function: ease-in-out;

/* discrete steps */
transition-timing-function: steps(8, end);

/* custom curve — visit cubic-bezier.com */
transition-timing-function: cubic-bezier(0.68, -0.55, 0.27, 1.55);

✅Tip: ease-out feels most natural for UI — fast start, soft landing.

CSS Animation

Define keyframes once, then play them with full control over timing, direction & repetition.

@keyframes

Definition
At-rule that defines the styles at each step of an animation.
Syntax
@keyframes name { 0% {…} 100% {…} }
Markers
from (=0%) and to (=100%), or any percentage.
@keyframes pulse {
  0%, 100% { transform: scale(1); }
  50%      { transform: scale(1.15); }
}
.box { animation: pulse 1.6s ease-in-out infinite; }

animation-name

References the @keyframes to use.

pulse
.box { animation-name: pulse; }

animation-duration

Length of one cycle.

3s
.box { animation-duration: 3s; }

animation-delay

Wait before starting.

delayed
.box { animation: fade 2s ease 0.5s infinite alternate; }

animation-direction

normal · reverse · alternate · alternate-reverse.

alternate
.box { animation-direction: alternate; }

animation-fill-mode

Which styles apply before/after — none · forwards · backwards · both.

both
.box { animation-fill-mode: both; }

animation-iteration-count

How many times to loop. Use infinite.

∞
.box { animation-iteration-count: infinite; }

animation-play-state

running · paused. Hover the spinner to pause.

​

Hover via .spinner:hover { animation-play-state: paused; }

.box { animation-play-state: running; }
.box:hover { animation-play-state: paused; }

animation-timing-function

Same curves as transitions — ease, steps(), cubic-bezier()…

bounce
.box { animation-timing-function: cubic-bezier(.5, 0, .5, 1); }

animation shorthand — order matters

/* name | duration | easing | delay | count | direction | fill-mode | play-state */
.box {
  animation: pulse 1.6s ease-in-out 0s infinite alternate both running;
}

⚠️Note: the first time value is duration, the second is delay. Mixing them up is the most common animation bug.
